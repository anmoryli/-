# 孕期宝差异化与幸福感升级计划

## 一、产品现状与市场问题

**当前定位**：记录 + AI 孕期咨询（健康顾问）

**市面同类产品**：大多以「孕产知识/咨询」为主，功能同质化严重（孕周、饮食、产检、胎动等）。

**你的优势**：已有完善的**多形态记录**（文字、照片、语音、文件、写信给宝宝）、PDF 导出、胎动/心情/宫缩、分享卡片。核心应是**记录与情感留存**，而非咨询。

---

## 二、核心竞争力重塑

### 1. 差异化定位


| 维度    | 市面上多数 App  | 孕期宝（升级后）            |
| ----- | ---------- | ------------------- |
| 主卖点   | 孕产知识、AI 问答 | **孕期记录与回忆留存**       |
| AI 角色 | 健康顾问、答疑    | **记录助手、回忆伙伴**       |
| 用户价值  | 获取信息       | **留存时光、情感陪伴、未来可回忆** |
| 核心情感  | 实用、工具感     | **温暖、专属、有归属感**      |


### 2. 一句话定位

> **孕期宝：专注孕期记录，把每一天都留下来，成为你和宝宝最珍贵的回忆。**

### 3. 四大差异化支柱

- **记录优先**：首页与导航以「记录」为主线，AI 为辅
- **情感价值**：写信给宝宝、心情、时光轴、回忆提炼
- **可带走**：PDF 导出、分享卡片、未来可做实体相册/书
- **简洁专注**：不堆医学科普，专注记录体验与幸福感

## 三、AI 对话重构：从咨询到记录助手

### 1. 角色转变


| 原定位        | 新定位          |
| ---------- | ------------ |
| 孕期健康顾问     | **记录陪伴助手**   |
| 解答饮食/产检等问题 | 辅助整理、回顾、提炼记录 |
| 医疗咨询风格     | 情感陪伴、回忆解读    |


### 2. 新 AI 能力（以记录为核心）

- **记录整理**：根据用户文字/照片/语音，生成「本周小结」「孕 X 周回顾」
- **写信助手**：帮用户润色给宝宝的信、补充温馨表达
- **回忆解读**：基于用户记录做情感/成长回顾（如「这周你记录了 X 次，心情以 X 为主」）
- **灵感提示**：推荐「今天可以记录什么」（如第一次胎动、产检感受）
- **弱化医疗**：医疗类问题统一引导「建议咨询医生」，仅做极简常识提醒

### 3. 前端与入口调整

- 底部导航：`AI助手` → `记录助手` 或 `小助手`
- 快捷操作：`问AI` → `记录灵感` 或 `写日记助手`
- 引导页 / 使用指南：去掉「解答孕期疑问」，改为「帮您整理记录、写信、回顾时光」
- 快捷问题：改为记录导向，例如：
  - 「帮我写一段本周小结」
  - 「这封信可以怎么改得更温馨」
  - 「今天有什么值得记录的」

### 4. 后端 Prompt 调整（[AiController.java](backend/src/main/java/com/anmory/yunji/controller/AiController.java)）

将 system prompt 从「孕期健康顾问」改为「记录陪伴助手」：

```
你是孕期宝的记录陪伴助手，温暖、细腻、有同理心。
你的主要职责是：
1. 帮助用户整理、回顾、提炼孕期记录（日记、照片、语音）
2. 辅助用户写信给宝宝，润色文字、补充温馨表达
3. 基于用户已有记录，生成「本周小结」「孕周回顾」等
4. 在用户不知道写什么时，给出记录灵感建议
5. 涉及饮食、产检、用药等医疗问题时，温和提醒「建议咨询医生」
请勿主动提供医疗建议，专注记录与情感陪伴。
```

## 四、新增幸福感功能（按优先级）

### P0：高价值、可快速落地


| 功能           | 说明                         | 幸福感来源         |
| ------------ | -------------------------- | ------------- |
| **孕期时光轴**    | 按时间线展示记录，支持按孕周筛选，可配合宝宝大小展示 | 直观看到「一路走来」的成长 |
| **每日/每周一句**  | 根据孕周或心情，推送一句温暖的话或小确幸       | 被看见、被鼓励       |
| **记录提醒（可选）** | 温和提醒「今天还没记录哟」，可设置时间        | 养成习惯、不遗漏      |


### P1：情感价值高


| 功能          | 说明                      | 幸福感来源      |
| ----------- | ----------------------- | ---------- |
| **胎动/心情趋势** | 简单图表展示胎动次数、心情变化         | 数据可视化带来成就感 |
| **「第一次」合集** | 自动识别并聚合「第一次胎动」「第一次产检」等  | 重要节点一目了然   |
| **给未来的自己**  | 写下给产后自己的话，设定「宝宝 X 岁」时再看 | 时间胶囊式期待    |


### P2：增强粘性


| 功能           | 说明                            | 幸福感来源   |
| ------------ | ----------------------------- | ------- |
| **孕期小目标**    | 如「每天记录 1 条」「写满 10 封信」等，达成有小奖励 | 成就感、仪式感 |
| **家人共享（可选）** | 准爸爸/家人可留言、点赞、补充记录             | 被家人关注   |
| **音乐/冥想**    | 孕期舒缓音乐、简短冥想引导（可外链或嵌入）         | 放松、减压   |

## 五、实施计划（分阶段）

### Phase 1：定位与 AI 重构（约 1–2 周）

1. 更新全站文案：欢迎页、引导页、使用指南、SEO
2. 修改 [AiController.java](backend/src/main/java/com/anmory/yunji/controller/AiController.java) system prompt
3. 修改 [chat/page.tsx](frontend/app/(app)/chat/page.tsx)：标题、描述、快捷问题
4. 修改 [quick-actions.tsx](frontend/components/home/quick-actions.tsx)：`问AI` → `记录灵感`
5. 修改 [bottom-nav.tsx](frontend/components/bottom-nav.tsx)：`AI助手` → `记录助手`

### Phase 2：幸福感 P0 功能（约 2–3 周）

1. **孕期时光轴**：新页面 `/timeline`，按日期/孕周展示记录，可筛选类型
2. **每日/每周一句**：扩展现有 `getWeeklyTip`，或新增 `getDailyWarmth` 接口
3. **记录提醒**：在通知设置中增加「每日记录提醒」开关与时间选择

### Phase 3：幸福感 P1 功能（约 2–3 周）

1. **胎动/心情趋势**：在首页或统计区增加简单折线图/柱状图
2. **「第一次」合集**：基于 tag 或关键词识别，在记录页增加「重要时刻」tab
3. **给未来的自己**：新记录类型 `letter_to_future`，支持设定「解锁时间」

### Phase 4：差异化打磨与 P2（可选）

1. 完善时光轴视觉、动画、分享
2. 孕期小目标、家人共享等按需求排期
3. 品牌与 App 商店描述统一为「记录优先、情感陪伴」

## 六、关键改点汇总


| 层级      | 改动                          |
| ------- | --------------------------- |
| **品牌**  | Slogan：「把每一天都留下来」；描述强调记录与回忆 |
| **首页**  | 记录相关模块更突出，AI 入口弱化或改为「记录灵感」  |
| **AI**  | 角色、Prompt、快捷问题全面转向记录助手      |
| **新页面** | 时光轴、给未来的自己、重要时刻合集           |
| **新能力** | 每日/每周一句、记录提醒、胎动心情趋势         |

## 七、预期效果

- **差异化**：市面少见「记录优先、AI 为辅」的孕期 App
- **情感价值**：从工具升级为「陪伴」和「回忆载体」
- **留存**：记录越多，迁移成本越高，粘性越强
- **传播**：PDF、分享卡片、时光轴截图更易形成传播
