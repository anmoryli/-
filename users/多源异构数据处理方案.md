## 一、现状分析

### 1.1 多源异构数据全景

孕期宝系统存在 **5 类异构数据源**，最终汇聚到 Milvus 向量库供 AI 对话检索：

```
┌────────────────────────────────────────────────────────────┐
│                     数据来源（异构）                         │
├──────────┬──────────┬──────────┬──────────┬────────────────┤
│ 文字/语音 │  照片    │ PDF 文件 │ 对话消息  │ 知识库百科      │
│ 记录      │ 记录     │ 记录     │          │ PDF/DOCX/MD   │
├──────────┴──────────┴──────────┴──────────┴────────────────┤
│                     处理层（异构策略）                       │
├──────────┬──────────┬──────────┬──────────┬────────────────┤
│ 标题+正文 │ 图像理解 │ PDFBox  │ 消息原文  │ 分块+清洗       │
│ +URL     │ +描述    │ 抽全文   │          │ 1500/100      │
│          │ +URL     │ +URL     │          │               │
├──────────┴──────────┴──────────┴──────────┴────────────────┤
│                 Milvus 向量库（统一存储）                    │
│          集合：yunji_rag / 1024维 / user_id 多租户           │
└────────────────────────────────────────────────────────────┘
```

### 1.2 当前数据流转路径

| 数据来源 | 写入 MySQL | 嵌入向量库 | 方式 |
|----------|-----------|-----------|------|
| 文字记录 | MemoController → memo + text | `ragService.embedAsync(userId, 标题+正文+URL, "memo", memoId)` | @Async 异步 |
| 语音记录 | MemoController → memo + voice | `ragService.embedAsync(userId, 标题+URL, "memo", memoId)` | @Async 异步 |
| 照片记录 | MemoController → memo + photo | 先调图像理解 → `ragService.embedAsync(userId, 描述+理解+URL, "image_desc", memoId)` | @Async 异步 |
| 文件/PDF 记录 | MemoController → memo + file | PDFBox 抽全文 → `ragService.embedAsync(userId, 正文+URL, "memo", memoId)` | CompletableFuture 异步 |
| 对话消息 | MessageService → message | `ragService.embedAsync(userId, content, "message", messageId)` | @Async 异步 |
| 产检 PDF | CheckReportService → check_report | PDFBox 抽全文 → `ragService.embedAsync(userId, text, "pdf", reportId)` | 同步后异步嵌入 |
| 知识库文档 | 不入 MySQL | Python RAG 服务分块后批量写入 Milvus (user_id=-1) | 独立流程 |

### 1.3 已发现的数据一致性问题

| # | 问题 | 影响 | 严重程度 |
|---|------|------|----------|
| 1 | **删除记录时向量库未同步清理** | 检索命中已删除内容，AI 回复引用"幽灵数据" | 🔴 高 |
| 2 | **嵌入失败静默丢失** | `embedAsync` 失败仅打日志，无重试，用户数据永远进不了向量库，检索缺失 | 🔴 高 |
| 3 | **编辑记录后向量库仍是旧内容** | 用户修改了标题/正文，但检索出来的是旧版本 | 🔴 高 |
| 4 | **异步时序竞争** | AI 生成标题（异步）与嵌入（异步）并行执行，嵌入时标题可能还是占位符 | 🟡 中 |
| 5 | **嵌入文本格式不统一** | 各处拼接文本方式不同，元数据缺失，检索精度受影响 | 🟡 中 |
| 6 | **无法审计嵌入状态** | 不知道哪些数据成功进了向量库、哪些丢了 | 🟡 中 |
| 7 | **无定期对账机制** | MySQL 与 Milvus 长期运行后数据漂移，无法自动发现和修复 | 🟡 中 |

```
│                        业务写入层                               │

                            │
                            ▼

│                    Milvus 向量库                                │

│  1. 回查 MySQL 确认记录存在                                     │

                            │

│                    数据对账任务 (每日)                           │
│  1. MySQL 有但向量库缺失 → 自动补建 embed_task                  │
│  2. 向量库有但 MySQL 已删除 → 调用删除接口清理                    │
│  3. 输出对账报告日志                                            │
└────────────────────────────────────────────────────────────────┘
```

## 三、具体改进方案

### 3.1 方案一：引入 embed_task 嵌入任务表

#### 3.1.1 建表

```sql
CREATE TABLE embed_task (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    source VARCHAR(20) NOT NULL COMMENT 'memo|message|pdf|image_desc',
    source_id VARCHAR(50) NOT NULL COMMENT '对应业务表主键',
    action VARCHAR(10) NOT NULL DEFAULT 'upsert' COMMENT 'upsert|delete',
    text_snapshot TEXT NULL COMMENT '嵌入时的完整文本快照（delete 时可为空）',
    status VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT 'pending|processing|success|failed',
    retry_count INT NOT NULL DEFAULT 0,
    max_retry INT NOT NULL DEFAULT 3,
    error_msg VARCHAR(500) NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status_created (status, created_at),
    INDEX idx_source (source, source_id),
    INDEX idx_user (user_id)
) COMMENT '向量库嵌入任务表，作为 MySQL 与 Milvus 之间的数据桥梁';
```

#### 3.1.2 状态机

```
        创建记录 / 编辑记录
               │
               ▼
           [pending]
               │
      定时任务扫描并锁定
               │
               ▼
          [processing]
            /       \
        成功          失败
         │             │
         ▼             ▼
      [success]    retry_count++
                       │
                  ┌────┴────┐
                  │ < 3次?  │
                  └────┬────┘
                  是    │    否
                  │     │     │
                  ▼     │     ▼
             [pending]  │  [failed] → 告警日志
                        │
      删除记录时
               │
               ▼
     插入 action=delete 的 pending 任务
               │
               ▼
     调度器执行 RAG 删除
               │
               ▼
           [success]
```

#### 3.1.3 核心代码逻辑

**业务层（替换直接调用 embedAsync）：**

```java
@Service
@RequiredArgsConstructor
public class EmbedTaskService {

    private final EmbedTaskMapper embedTaskMapper;

    /**
     * 提交嵌入任务（新增/编辑记录时调用）
     * 若同一 source+source_id 已有 pending 任务，则更新文本快照
     */
    public void submitUpsert(int userId, String text, String source, String sourceId) {
        EmbedTask existing = embedTaskMapper.selectPendingBySource(source, sourceId);
        if (existing != null) {
            // 更新快照（可能是 AI 生成标题后的二次提交）
            embedTaskMapper.updateTextSnapshot(existing.getId(), text);
            return;
        }
        EmbedTask task = new EmbedTask();
        task.setUserId(userId);
        task.setSource(source);
        task.setSourceId(sourceId);
        task.setAction("upsert");
        task.setTextSnapshot(text);
        task.setStatus("pending");
        embedTaskMapper.insert(task);
    }

    /**
     * 提交删除任务（删除记录时调用）
     */
    public void submitDelete(int userId, String source, String sourceId) {
        EmbedTask task = new EmbedTask();
        task.setUserId(userId);
        task.setSource(source);
        task.setSourceId(sourceId);
        task.setAction("delete");
        task.setStatus("pending");
        embedTaskMapper.insert(task);
    }
}
```

**嵌入调度器（替换 @Async）：**

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class EmbedTaskRunner {

    private final EmbedTaskMapper embedTaskMapper;
    private final RagService ragService;

    @Scheduled(fixedDelay = 10_000)  // 每 10 秒扫描
    public void processPendingTasks() {
        List<EmbedTask> tasks = embedTaskMapper.selectPending(50);  // 批量取 50 条
        for (EmbedTask task : tasks) {
            try {
                embedTaskMapper.updateStatus(task.getId(), "processing");

                if ("delete".equals(task.getAction())) {
                    ragService.deleteBySourceId(task.getSource(), task.getSourceId());
                } else {
                    String text = task.getTextSnapshot();
                    if (text == null || text.isBlank()) {
                        log.warn("嵌入任务文本为空，跳过 taskId={}", task.getId());
                        embedTaskMapper.updateStatus(task.getId(), "failed");
                        continue;
                    }
                    ragService.embedSync(task.getUserId(), text,
                                         task.getSource(), task.getSourceId());
                }

                embedTaskMapper.updateStatus(task.getId(), "success");

            } catch (Exception e) {
                int newRetry = task.getRetryCount() + 1;
                if (newRetry >= task.getMaxRetry()) {
                    embedTaskMapper.markFailed(task.getId(), newRetry, e.getMessage());
                    log.error("[嵌入] 任务最终失败 taskId={} source={} sourceId={}",
                              task.getId(), task.getSource(), task.getSourceId(), e);
                } else {
                    embedTaskMapper.markRetry(task.getId(), newRetry, e.getMessage());
                    log.warn("[嵌入] 任务重试 {}/{} taskId={}",
                             newRetry, task.getMaxRetry(), task.getId());
                }
            }
        }
    }
}
```

### 3.2 方案二：向量库增加删除能力

#### 3.2.1 RAG 服务新增删除接口（Python 端）

```python
@app.delete("/api/v1/delete")
async def delete_vectors(source: str, source_id: str):
    """按 source + source_id 删除向量库中的对应条目"""
    expr = f'source == "{source}" and source_id == "{source_id}"'
    collection.delete(expr)
    return {"code": 200, "message": "deleted"}
```

#### 3.2.2 Java 端 RagService 新增

```java
/**
 * 同步嵌入（供调度器使用，非 @Async）
 */
public void embedSync(int userId, String text, String source, String sourceId)
        throws IOException {
    // 与 embedAsync 逻辑相同，但同步执行，异常向上抛出
    String body = GSON.toJson(Map.of(
        "user_id", userId,
        "text", text.trim().length() > 10000 ? text.trim().substring(0, 10000) : text.trim(),
        "source", source != null ? source : "memo",
        "source_id", sourceId != null ? sourceId : ""
    ));
    Request request = new Request.Builder()
        .url(BASE_URL + "/api/v1/embed")
        .post(RequestBody.create(body, JSON))
        .build();
    try (Response response = CLIENT.newCall(request).execute()) {
        if (!response.isSuccessful()) {
            throw new IOException("RAG embed 失败 code=" + response.code());
        }
    }
}

/**
 * 按 source + source_id 删除向量
 */
public void deleteBySourceId(String source, String sourceId) throws IOException {
    String url = BASE_URL + "/api/v1/delete?source="
        + URLEncoder.encode(source, StandardCharsets.UTF_8)
        + "&source_id=" + URLEncoder.encode(sourceId, StandardCharsets.UTF_8);
    Request request = new Request.Builder().url(url).delete().build();
    try (Response response = CLIENT.newCall(request).execute()) {
        if (!response.isSuccessful()) {
            throw new IOException("RAG delete 失败 code=" + response.code());
        }
    }
}
```

### 3.3 方案三：统一嵌入文本格式

#### 3.3.1 格式规范

所有写入向量库的文本统一为以下结构化模板：

```
[来源:{source}] [类型:{type}] [孕周:{week}周] [日期:{date}] [心情:{mood}]
标题：{title}
正文：{content}
[标签:{tags}]
[URL] {url}
```

#### 3.3.2 格式化工具类

```java
public class EmbedTextFormatter {

    /**
     * 统一格式化嵌入文本
     */
    public static String format(EmbedContext ctx) {
        StringBuilder sb = new StringBuilder();
        // 元数据行
        sb.append("[来源:").append(ctx.source).append("]");
        sb.append(" [类型:").append(ctx.type).append("]");
        if (ctx.pregnancyWeek != null)
            sb.append(" [孕周:").append(ctx.pregnancyWeek).append("]");
        if (ctx.date != null)
            sb.append(" [日期:").append(ctx.date).append("]");
        if (ctx.mood != null && !ctx.mood.isBlank())
            sb.append(" [心情:").append(ctx.mood).append("]");
        sb.append("\n");
        // 标题
        if (ctx.title != null && !ctx.title.isBlank())
            sb.append("标题：").append(ctx.title).append("\n");
        // 正文
        if (ctx.content != null && !ctx.content.isBlank())
            sb.append("正文：").append(ctx.content).append("\n");
        // 标签
        if (ctx.tags != null && !ctx.tags.isBlank())
            sb.append("[标签:").append(ctx.tags).append("]\n");
        // URL
        if (ctx.url != null && !ctx.url.isBlank())
            sb.append("[URL] ").append(ctx.url).append("\n");
        // 截断
        String result = sb.toString().trim();
        return result.length() > 10000 ? result.substring(0, 10000) : result;
    }

    @lombok.Builder
    @lombok.Data
    public static class EmbedContext {
        private String source;       // memo / message / pdf / image_desc
        private String type;         // text / voice / photo / file / chat / report
        private String pregnancyWeek;
        private String date;
        private String mood;
        private String title;
        private String content;
        private String tags;
        private String url;
    }
}
```

### 3.4 方案四：解决异步时序竞争

#### 3.4.1 问题根因

```
当前时序（有 Bug）:
  t0: 写入 MySQL（标题 = 占位符 "记录"）
  t1: 同时异步 → AI 生成标题          → t3 回写 MySQL（标题 = "第一次产检"）
  t2: 同时异步 → embedAsync（用占位符嵌入）  ← 嵌入的是错误内容！
```

#### 3.4.2 修复方案

使用 embed_task 表的延迟嵌入机制：

```
修复后时序:
  t0: 写入 MySQL（标题 = 占位符）
  t1: 创建 embed_task（status = 'waiting_enrich'，暂不执行嵌入）
  t2: 异步 AI 生成标题/分类 → 回写 MySQL
  t3: AI 丰富化回调 → 用最新内容更新 embed_task.text_snapshot
                    → 状态改为 'pending'
  t4: 调度器扫描到 pending → 用正确内容嵌入
```

embed_task 表增加一个状态：

```sql
-- status 增加 'waiting_enrich' 状态
-- waiting_enrich → pending → processing → success/failed
```

**MemoAiEnrichmentService 回调中触发：**

```java
// AI 丰富化完成后
public void onEnrichmentComplete(Integer memoId) {
    Memo memo = memoMapper.selectById(memoId);
    Text text = textMapper.selectByMemoId(memoId);
    // 用最新的标题+正文+分类重新格式化
    String embedText = EmbedTextFormatter.format(
        EmbedContext.builder()
            .source("memo").type(memo.getType())
            .title(text.getTitle()).content(text.getContent())
            .pregnancyWeek(memo.getPregnancyWeek())
            .mood(memo.getMood()).tags(memo.getCategory())
            .build()
    );
    // 更新 embed_task 文本并激活
    embedTaskService.activateTask("memo", String.valueOf(memoId), embedText);
}
```

### 3.5 方案五：编辑记录时同步更新向量库

```
编辑记录
    │
    ├── 1. 更新 MySQL（memo + text/voice/file/photo）
    │
    ├── 2. 提交删除旧向量任务
    │      embedTaskService.submitDelete(userId, "memo", memoId)
    │
    └── 3. 提交新嵌入任务（用最新内容）
           embedTaskService.submitUpsert(userId, newText, "memo", memoId)
```

由于 embed_task 调度器按创建时间顺序执行，delete 会先于 upsert 执行，保证先删后增。

### 3.6 方案六：检索结果回查 MySQL 校验

#### 3.6.1 校验逻辑

在 AI 对话拼接 context 之前，对 RAG 返回的每条 hit 做存在性 + 权限校验：

```java
public List<RagHit> validateHits(List<RagHit> hits, Integer currentUserId) {
    return hits.stream().filter(hit -> {
        String source = hit.getSource();
        String sourceId = hit.getSourceId();

        // 百科类数据（user_id = -1）不需要回查
        if (hit.getUserId() != null && hit.getUserId() == -1) return true;

        switch (source) {
            case "memo":
            case "image_desc":
                Memo memo = memoMapper.selectById(Integer.parseInt(sourceId));
                if (memo == null) {
                    log.debug("[检索校验] memo 已删除 sourceId={}", sourceId);
                    return false;
                }
                // 校验可见性权限
                if (!isVisibleToUser(memo, currentUserId)) {
                    log.debug("[检索校验] memo 对当前用户不可见 sourceId={}", sourceId);
                    return false;
                }
                return true;

            case "message":
                Message msg = messageMapper.selectById(Integer.parseInt(sourceId));
                if (msg == null) return false;
                // 消息所属会话必须是当前用户的
                Conversation conv = conversationMapper.selectById(msg.getConversationId());
                return conv != null && conv.getUserId().equals(currentUserId);

            case "pdf":
                CheckReport report = checkReportMapper.selectById(Integer.parseInt(sourceId));
                if (report == null) return false;
                return report.getUserId().equals(currentUserId);

            default:
                return true;
        }
    }).toList();
}
```

#### 3.6.2 性能优化

回查可能带来 N+1 查询问题，优化方案：

```java
// 批量查询：先按 source 分组，批量查 MySQL，再做过滤
Map<String, List<RagHit>> bySource = hits.stream()
    .collect(Collectors.groupingBy(RagHit::getSource));

// 批量查 memo
List<Integer> memoIds = bySource.getOrDefault("memo", List.of()).stream()
    .map(h -> Integer.parseInt(h.getSourceId())).toList();
Map<Integer, Memo> memoMap = memoMapper.selectByIds(memoIds).stream()
    .collect(Collectors.toMap(Memo::getMemoId, m -> m));

// 然后逐条校验时从 Map 中取，避免逐条查库
```

### 3.7 方案七：数据对账任务

#### 3.7.1 定时对账调度器

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class EmbedReconciliationScheduler {

    private final EmbedTaskMapper embedTaskMapper;
    private final MemoMapper memoMapper;
    private final MessageMapper messageMapper;
    private final CheckReportMapper checkReportMapper;
    private final EmbedTaskService embedTaskService;
    private final RagService ragService;

    @Scheduled(cron = "0 0 3 * * ?")  // 每天凌晨 3 点
    public void reconcile() {
        log.info("[对账] 开始 MySQL ↔ Milvus 数据对账");

        int missing = 0, orphan = 0;

        // === 1. 检查 MySQL 中有但向量库中缺失的数据 ===

        // 查所有应该在向量库中的 memo
        List<Integer> allMemoIds = memoMapper.selectAllIds();
        // 查 embed_task 中 source=memo 且 status=success 的 source_id
        Set<String> embeddedMemoIds = embedTaskMapper
            .selectSuccessSourceIds("memo").stream().collect(Collectors.toSet());

        for (Integer memoId : allMemoIds) {
            if (!embeddedMemoIds.contains(String.valueOf(memoId))) {
                // 缺失：自动补建嵌入任务
                Memo memo = memoMapper.selectById(memoId);
                if (memo != null) {
                    String text = buildEmbedText(memo);  // 格式化
                    embedTaskService.submitUpsert(memo.getUserId(), text,
                                                  "memo", String.valueOf(memoId));
                    missing++;
                }
            }
        }

        // === 2. 检查向量库中有但 MySQL 已删除的脏数据 ===

        Set<String> allMemoIdSet = allMemoIds.stream()
            .map(String::valueOf).collect(Collectors.toSet());

        for (String embeddedId : embeddedMemoIds) {
            if (!allMemoIdSet.contains(embeddedId)) {
                // 脏数据：提交删除任务
                embedTaskService.submitDelete(0, "memo", embeddedId);
                orphan++;
            }
        }

        log.info("[对账] 完成：补建 {} 条缺失嵌入，清理 {} 条脏数据", missing, orphan);
    }
}
```
