# 孕期宝 需求文档

## 一、需求分析

### 1.1 项目背景与目标

孕期宝是一款面向孕妇及家庭的孕期记录与陪伴应用。目标用户包括：孕妇本人、配偶及其他家庭成员。产品目标为：

- **记录留存**：帮助用户以文字、语音、照片、文件等形式记录孕期点滴，形成可导出、可分享的时光轴与纪念册。
- **智能陪伴**：通过 AI 对话提供记录整理、写信给宝宝、本周小结、情绪共情、定时提醒等能力，提升情绪价值。
- **家庭协同**：支持家庭组建、成员邀请、记录共享、评论点赞、爸爸成长营任务分配与站内通知，增强家庭参与感。
- **健康与目标**：提供每日胎动/心情/体重/健康值、宫缩记录、产检单上传与解析、孕期小目标与成就，在非医疗前提下辅助用户自我管理。

### 1.2 用户角色

| 角色 | 说明 | 典型能力 |
|------|------|----------|
| 孕妇 | 主账号，创建家庭 | 全量记录、AI 对话、图生图、目标与成就、产检单、分享范围设置 |
| 家庭成员 | 通过邀请码加入家庭 | 查看共享记录、评论点赞、家庭任务、配偶可使用 AI 对话与图生图 |
| 管理员 | 后台 | 用户与内容管理、科普文章发布、数据统计 |

### 1.3 核心场景

1. **记录与回顾**：写日记、拍照片、录语音、上传文件；按时间轴/类型/标签筛选；导出 PDF 纪念册或日期范围 PDF。
2. **AI 交互**：多轮对话、流式回复；基于向量检索的“找图/搜记录”；写信给宝宝、本周小结、记录灵感、情绪共情；定时提醒（最多 2 轮收集）；图生图与社区模板。
3. **家庭互动**：创建/加入家庭、维护成员关系与配偶标识；记录可见范围（全部/白名单/黑名单）；评论、点赞；爸爸成长营任务建议与分配、站内通知。
4. **每日与健康**：胎动、心情、体重、健康值；宫缩记录；产检单上传与 AI 解析、复查日期提醒。
5. **目标与成就**：孕期小目标模板、用户进度、成就徽章。

### 2.1 用户与账号

#### U01 注册/登录

- **功能描述**：用户通过用户名、密码完成注册或登录；登录/注册时可要求输入图形验证码，防止暴力尝试。
- **使用流程**：  
  1. 用户打开登录页，输入用户名、密码；若开启验证码则先获取并填写验证码。  
  2. 点击「登录」：系统校验用户名与密码（及验证码），通过则生成会话并返回 token/用户信息，前端跳转至首页；失败则提示错误信息。  
  3. 未注册用户可点击「注册」：进入注册页，填写用户名、密码、确认密码；可选填与孕妇关系（用于后续加入家庭）；提交后系统校验用户名是否重复、密码强度等，通过则创建账号并视为已登录。
- **业务规则**：密码需加密存储；验证码需服务端校验并在一定时间内有效。

#### U02 个人资料

- **功能描述**：用户可维护头像、昵称、邮箱，修改密码，以及通过邮箱找回密码。
- **使用流程**：  
  1. **头像**：进入个人资料/设置，点击头像区域上传图片；系统上传至 OSS 并更新 `user.avatar_url`，前端刷新展示。  
  2. **昵称**：在资料页修改昵称并保存；系统更新 `user.username`。  
  3. **邮箱绑定**：输入邮箱并保存；用于接收产检提醒、配偶提及等邮件及找回密码。  
  4. **修改密码**：在安全设置中输入当前密码与新密码；系统校验当前密码后更新密码哈希。  
  5. **找回密码**：在登录页点击「忘记密码」，输入已绑定邮箱；系统发送带重置链接的邮件（若实现）；用户通过链接设置新密码。
- **业务规则**：修改敏感信息时需校验当前身份（登录态）。

#### U03 孕产信息

- **功能描述**：孕妇可填写末次月经日、预产期，用于计算孕周、宝宝发育描述等；家庭成员可为空。
- **使用流程**：  
  1. 进入个人资料或孕产信息页。  
  2. 选择/填写末次月经日、预产期，保存。  
  3. 系统按末次月经日计算当前孕周（如「孕 12 周」），在时间轴、每日日志、AI 能力等处展示。
- **业务规则**：孕妇账号建议必填；家庭成员可不填，部分依赖孕周的能力则使用默认或隐藏。

#### U04 用户类型

- **功能描述**：区分孕妇（pregnant）与家庭成员（family_member）；注册时可选「与孕妇关系」，便于加入家庭时预填。
- **使用流程**：  
  1. 注册时选择身份：孕妇本人 / 家庭成员；若选家庭成员可再选关系（如配偶、婆婆、妈妈等）。  
  2. 系统将身份写入 `user.user_type`、关系写入 `user.default_relationship`。  
  3. 加入家庭时若未填关系，可再次填写；配偶关系会用于开放 AI 对话、图生图等权限判断。
- **业务规则**：仅孕妇本人或被识别为配偶的家庭成员可使用 AI 对话与图生图。

#### U05 隐私与数据

- **功能描述**：用户可选择是否允许匿名使用数据被收集以改进服务（如统计、分析），默认关闭。
- **使用流程**：在设置/隐私页查看并切换「允许匿名数据收集」开关；系统更新 `user.data_collection_enabled`。
- **业务规则**：关闭时不将可识别用户的数据用于非必要分析。

#### U06 分享范围

- **功能描述**：孕妇可设置对家人共享的记录范围：全部 / 仅信件（如 letter_to_baby）/ 仅照片。
- **使用流程**：在家庭或共享设置页选择「全部」「仅信件」「仅照片」之一并保存；系统更新 `user.share_scope`。家庭成员在「家人的记录」中仅能查看落在该范围内的记录。
- **业务规则**：仅孕妇账号可修改；家庭成员视图按该范围过滤。

### 2.2 孕期记录（备忘录）

#### M01 文字记录

- **功能描述**：用户可新增一条文字记录，包含标题、正文、孕周、标签（如 letter_to_baby）、心情、可见范围、分类；支持由 AI 生成标题与分类（可异步补齐）。
- **使用流程**：  
  1. 用户进入「写日记」或「写信给宝宝」等入口，进入编辑页。  
  2. 填写标题（可留空由 AI 生成）、正文；可选填孕周、标签、心情、可见范围（全部/仅指定成员可见/仅指定成员不可见）、分类。  
  3. 提交后，系统先落库（memo + text），再异步：若标题或分类为空则调用 AI 生成并回写；同时将「标题+正文」及可选 `[URL]` 写入向量库（source=memo），便于后续对话检索。  
  4. 用户可在时间轴中看到该条记录，支持按类型/标签/关键词筛选。
- **业务规则**：正文必填；可见范围与「分享范围」共同作用；单条记录对应向量库一条嵌入（不分块）。

#### M02 语音记录

- **功能描述**：用户上传语音文件得到 URL，并填写标题、孕周、心情、可见范围、分类；系统可异步生成标题或分类。
- **使用流程**：  
  1. 用户选择「录语音」，录制或选择语音文件并上传。  
  2. 系统将语音上传至 OSS，得到 URL，并写入 memo + voice 表。  
  3. 用户填写标题（可留空）、孕周、心情、可见范围、分类后保存。  
  4. 若标题或分类为空，后台异步调用 AI 生成并回写；并将「标题 + 可选描述」及 `[URL]` 写入向量库（source=memo）。  
  5. 时间轴展示该条语音记录，可点击播放。
- **业务规则**：语音 URL 必填；向量嵌入内容为文本摘要或标题，不含语音转写（若未实现转写）。

#### M03 照片记录

- **功能描述**：用户上传一张或多张照片（最多 9 张），可填标题、描述、孕周、心情、可见范围、分类；系统先调用图像理解模型得到描述，再与用户描述拼接后异步写入向量库，并带图片 URL。
- **使用流程**：  
  1. 用户选择「拍照片」，选择或拍摄多张图片上传。  
  2. 系统上传至 OSS，得到多个 URL，写入 memo + photo 等表；用户可填写标题、文字描述、孕周、心情、可见范围、分类。  
  3. 提交后，系统先保存记录并返回成功；后台异步：对首图调用图像理解模型，将「用户描述 + 图像理解结果」拼成固定句式，并追加 `[URL]` 后调用 RAG 嵌入（source=image_desc），便于后续「找图」类检索。  
  4. 时间轴展示缩略图与标题，点击可查看大图及描述。
- **业务规则**：照片至少一张；向量库中保留图片 URL，便于 AI 回复时输出 `![描述](url)` 供前端渲染。

#### M04 文件记录

- **功能描述**：用户上传文件（如 PDF），可填标题、孕周、心情、可见范围、分类；若为 PDF，系统会异步抽取正文并连同文件 URL 写入向量库；非 PDF 则用标题 + URL 嵌入。
- **使用流程**：  
  1. 用户选择「上传文件」，选择本地文件并上传。  
  2. 系统上传至 OSS，得到 URL，写入 memo + file；若未填标题则可由 AI 根据文件名生成。  
  3. 若文件为 PDF：后台异步使用 PDFBox 抽取全文，将「抽取正文 + [URL] 文件链接」写入向量库（source=memo）；若非 PDF，则将「标题 + [URL]」写入向量库。  
  4. 时间轴展示文件标题与类型，用户可点击打开或下载。
- **业务规则**：PDF 正文抽取在 Java 侧完成，整篇（或截断）作为一条文本送入 RAG 嵌入，不在 RAG 服务内再分块。

#### M05 时间轴展示

- **功能描述**：按日期分组展示用户（或所查看家人）的记录，支持按类型、标签、重要时刻、给未来的自己等筛选，以及关键词搜索；展示孕周、体重等辅助信息；标签超过 6 字时截断并省略号显示。
- **使用流程**：  
  1. 用户进入「孕期时光轴」页；若为家庭成员则可选查看「自己的」或「家人的」记录（受分享范围与可见范围约束）。  
  2. 顶部可选择筛选：全部、重要时刻、给未来的自己、文字、照片、语音、文件；搜索框可输入关键词，在标题、内容、描述、标签中检索。  
  3. 列表按日期倒序分组，每组显示日期、孕周、体重（若有）；每条显示类型图标、标签（≤6 字展示，超出截断）、心情、标题与内容摘要、图片缩略图（若有）、时间。  
  4. 点击某条进入详情或编辑。
- **业务规则**：可见性由 memo.visibility_mode、visible_to 与 user.share_scope 共同决定；标签来自 category，展示时做长度与逗号分隔处理。

#### M06 记录编辑与删除

- **功能描述**：用户可对本人记录进行编辑（标题、内容、孕周、心情、可见范围、分类等）或删除。
- **使用流程**：  
  1. 在时间轴或详情页点击「编辑」，进入编辑表单，修改后保存；系统更新 memo 及关联的 text/voice/file/photo。  
  2. 点击「删除」并确认后，系统软删或物理删除该 memo 及子表记录；向量库中对应 source_id 的条目不会自动删除，依赖后续重建或按 source_id 清理策略（若实现）。
- **业务规则**：仅记录所属用户可编辑/删除；家庭成员无编辑权限。

#### M07 可见范围

- **功能描述**：单条记录可设置「全部可见」「仅指定成员可见」「仅指定成员不可见」（allowlist/blocklist），与账号级「分享范围」一起决定家庭成员能看到哪些记录。
- **使用流程**：  
  1. 新增或编辑记录时，在「可见范围」中选择：全部 / 仅以下成员可见 / 仅以下成员不可见，并勾选具体成员（或留空表示全部）。  
  2. 系统将选择结果写入 memo.visibility_mode 与 memo.visible_to。  
  3. 家庭成员在查看家人记录时，系统按 share_scope 与每条记录的 visibility_mode、visible_to 过滤。
- **业务规则**：allowlist 下 visible_to 为空可视为仅孕妇本人可见；blocklist 下为空表示所有人可见。

#### M08 导出

- **功能描述**：支持将记录导出为 PDF 纪念册（含封面、按孕周/日期分组、结语寄语）；或按日期范围导出该时间段内的记录为 PDF。
- **使用流程**：  
  1. 用户进入时间轴或导出入口，选择「导出全部」或「按日期范围导出」。  
  2. 若选日期范围，则选择起止日期后确认。  
  3. 系统汇总记录，调用 PDF 导出服务生成 PDF（含 AI 结语、封面、分组、图片与链接等），返回文件流或下载链接。  
  4. 用户保存或打开 PDF。
- **业务规则**：导出内容受当前用户身份与可见范围约束；AI 结语由提示词模板与记录内容生成。

### 2.3 AI 对话与能力

#### A01 会话管理

- **功能描述**：用户可创建新会话、查看会话列表、删除会话、进入某会话后加载历史消息。
- **使用流程**：  
  1. 进入 AI 对话页：若无会话则展示「开始新对话」；若有则展示会话列表（标题、更新时间）。  
  2. 点击「开始新对话」：系统调用创建会话接口，生成新 conversation，前端进入该会话的聊天视图，消息列表为空。  
  3. 点击列表中某条会话：系统按 conversationId 拉取该会话下全部 message，按时间序展示；用户可继续在该会话中发消息。  
  4. 在会话列表中对某条会话执行删除：系统删除该 conversation 及其下所有 message，前端从列表移除并若当前正在该会话则清空消息区。
- **业务规则**：会话与 message 均按 userId 鉴权；会话标题可由首条用户消息截断或由后端生成。

#### A02 流式对话

- **功能描述**：用户发送文本（及可选图片）后，AI 以 SSE 流式逐字返回回复；多轮对话会带入近期历史上下文。
- **使用流程**：  
  1. 用户在输入框输入文字，可附一张或多张图片，点击发送。  
  2. 前端将内容与图片以 FormData 提交至流式接口；若当前无会话则先创建会话再发送。  
  3. 后端进行意图识别后，若非提醒/图生图等分支，则先保存用户消息（且本轮不嵌入向量库），再调用 RAG 检索，将检索结果拼入 system prompt，再调用大模型流式生成。  
  4. 前端通过 SSE 逐块接收内容并追加到当前 AI 消息气泡，流结束标志（done）后停止；同时保存 AI 回复到 message 表，并处理配偶提及、通知等。  
  5. 用户可继续输入，形成多轮；每轮都会带近期对话历史与 RAG 结果（若有）。
- **业务规则**：流式超时（如 2 分钟）后前端中止请求并恢复发送按钮；发送按钮在未登录或无权限时禁用。

#### A03 意图识别

- **功能描述**：系统根据用户输入与是否带图，识别为：普通聊天、找图/搜记录、写信给宝宝、图生图、图片理解、定时提醒等，并据此走不同分支（RAG 检索、图生图、提醒多轮收集等）。
- **使用流程**：  
  1. 用户发送一条消息（可能带图）后，后端首先调用意图识别（如 LLM 或规则）：输出 TEXT_CHAT、TEXT_TO_IMAGE、IMAGE_TO_IMAGE、IMAGE_UNDERSTANDING、REMINDER 之一。  
  2. 若为 REMINDER：进入定时提醒多轮收集流程，不执行 RAG。  
  3. 若为 TEXT_TO_IMAGE/IMAGE_TO_IMAGE：进入图生图流程，使用模板或用户提示词生成图片。  
  4. 若为 IMAGE_UNDERSTANDING 或 TEXT_CHAT：进入普通对话分支；其中「找图/搜图/查图」类必须判为 TEXT_CHAT 或 IMAGE_UNDERSTANDING，不判为图生图，确保走 RAG 检索而非生成。  
  5. 用户无感知，仅发现系统行为符合预期（如说「找之前的玉桂狗照片」会检索并展示图片，而不会去生成图）。
- **业务规则**：找/搜/查 + 无「生成/画图」等词时，强制不为 *_TO_IMAGE；有图且未要求生成时，一般为 IMAGE_UNDERSTANDING。

#### A04 RAG 检索

- **功能描述**：除纯提醒分支外，每轮对话会以用户问题（及经图像理解后的综合描述）向向量库检索 top_k 条相关片段，注入 system prompt，并要求模型在回复中保留图片的 Markdown 图片语法、文件的链接；必要时从 RAG 结果中解析图片 URL 追加到回复末尾，保证前端能渲染。
- **使用流程**：  
  1. 用户发送问题（如「我之前写的那段关于产检的心情在哪」）。  
  2. 后端将问题（若有图则先做图像理解再拼接）调用 RAG 服务 `/api/v1/query`，传入 userId、include_global=true、top_k=10。  
  3. RAG 服务将问题向量化，在 yunji_rag 中检索用户私有 + 百科（user_id=-1），按相似度排序返回 hits。  
  4. 后端将 hits 拼成上下文写入 system prompt，并约定：参考内容中的图片必须用 `![描述](url)`、文件用 `[点击查看](url)`；若模型未输出检索到的图片，则从上下文中解析 URL 追加到回复末尾。  
  5. 前端收到回复后，用 Markdown 组件渲染，图片与链接可点击或展示。
- **业务规则**：检索轮次不将用户问题写入向量库（embedInRag=false）；RAG 失败时仅打日志，对话继续无上下文回复。

#### A05 检索不嵌入

- **功能描述**：当本轮对话会执行 RAG 检索（从向量库「找东西」）时，不把当前用户问题写入向量库，避免「找图/搜记录」类查询污染检索集。
- **使用流程**：系统在流式主路径保存用户消息时调用 `MessageService.save(..., embedInRag=false)`；提醒分支保存用户消息时传入 `embedInRag=true`。用户无直接操作，仅体现为检索结果更干净、少无关命中。
- **业务规则**：仅「会执行 RAG 检索」的轮次不嵌入用户消息；非流式 /chat 未走 RAG，仍按原逻辑可嵌入（若实现）。

#### A06 写信给宝宝

- **功能描述**：当用户表达「写信给宝宝」「帮我想一句对宝宝说的话」等意图时，系统在 system 中注入写信引导 prompt，提供结构建议与润色，并可结合用户已有记录。
- **使用流程**：  
  1. 用户输入如「帮我写一段给宝宝的心里话」或点击快捷问题「帮我想一句对宝宝说的话」。  
  2. 后端检测到关键词（写信 + 宝宝/给宝宝/对宝宝），则在拼接 system 时追加 letter_to_baby_guide 的 system 文案。  
  3. 模型按引导生成或润色一段「写给宝宝」的短文字，用户可复制到记录或继续修改。  
  4. 用户也可在记录中先写草稿再在对话中请求「把这句改得更温馨」等，同样走写信引导逻辑。
- **业务规则**：引导仅增加 prompt，不强制保存到记录；用户可自行复制到文字记录并打 tag=letter_to_baby。

#### A07 情绪共情

- **功能描述**：当用户表达疲惫、焦虑、开心、委屈等情绪时，AI 先简短共情再给建议或记录灵感；涉及累/烦/担心/开心/紧张等词时优先情感支持。
- **使用流程**：用户输入如「今天好累」「有点担心产检」等，后端在 system 中已包含情绪与共情规则，模型会先回应共情再给建议或推荐写一条记录；用户无额外操作。
- **业务规则**：通过 prompt 约束，不单独做情绪分类接口；医疗建议仍提醒「建议咨询医生」。

#### A08 每日暖心语

- **功能描述**：在聊天页无消息时，展示一句基于当前孕周与日期的暖心语/鼓励语，由独立接口生成。
- **使用流程**：  
  1. 用户进入聊天页且当前会话无任何消息时，前端请求 `GET /api/ai/daily-warm?userId=xx`。  
  2. 后端根据用户孕周与当前日期调用 daily_warm_message 提示词，生成一句简短暖心语并返回。  
  3. 前端在空状态区以斜体等形式展示该句；若接口失败则展示默认句「今天也要好好照顾自己呀～」。
- **业务规则**：仅无消息时请求；有消息后不再展示该句。

#### A09 配偶端

- **功能描述**：当当前用户为家庭成员且被识别为配偶时，AI 使用配偶专属 system prompt：侧重孕期知识、如何照顾妻子、情感支持与沟通话术、小惊喜建议等，并继续强调医疗问题建议咨询医生。
- **使用流程**：配偶用户登录后进入 AI 对话，发送任意消息；后端根据 userId 判断为配偶后加载 ai_chat_system_spouse，模型回复风格与内容侧重准爸爸角色；用户无额外设置。
- **业务规则**：配偶身份由家庭成员关系 + LLM 异步判断的 is_spouse 或关系关键词决定。

#### A10 定时提醒

- **功能描述**：用户通过自然语言添加提醒（如「明天下午 3 点去人民公园散步」），系统最多 2 轮收集时间、地点、做什么、备注，解析为 once 或 daily，写入 scheduled_operation；入库后校验存在再回复，并附带表格；到期触发时内容经 LLM 改写后发送站内通知与邮件。
- **使用流程**：  
  1. 用户输入如「明天下午 3 点提醒我去人民公园散步」。  
  2. 后端意图识别为 REMINDER，进入多轮收集：第一轮尽量从一句话中抽全 time/place/what，不足则追问一句（如「几点？去哪儿？做啥？」）；第二轮必须合并已有状态并 complete，缺项用默认值写入。  
  3. 系统解析时间（含 schedule_parse 兜底）、写入 scheduled_operation，再 selectById 校验存在后，回复成功并附带 Markdown 表格（时间、地点、做什么、备注、类型）。  
  4. 到设定时间后，定时任务扫描到期记录，调用 ReminderRewriteService 将原始内容改写为「当前场景」话术，再发站内通知与邮件（若用户绑定邮箱）。
- **业务规则**：最多 2 轮；scheduleType 默认 once，仅明确「每天」且给时间时为 daily；触发文案必须改写后下发。

#### A11 图生图与社区

- **功能描述**：用户可保存提示词为模板（公开/私有）、基于模板或自由提示词 + 一张输入图生成新图；作品可公开到社区；社区内支持对作品的点赞与评论（含多层回复与评论点赞）。
- **使用流程**：  
  1. **模板**：用户在模板页创建模板（标题、提示词、分类、封面图），可选择是否公开；可查看公开模板与自己的模板。  
  2. **图生图**：用户上传一张图并输入提示词（或选择模板），勾选是否公开到社区后提交；后端调用图像生成 API，将结果图上传 OSS，写入 ai_generation_post，若公开则社区可见。  
  3. **社区**：用户浏览公开作品列表/推荐列表，可对作品点赞、评论；评论可回复他人（parent_comment_id）；可对评论点赞。  
  4. **我的作品**：用户可查看自己生成的作品列表，并切换某条的公开/私有状态。
- **业务规则**：仅孕妇或配偶可使用图生图；公开作品会展示提示词、输入图、输出图；点赞与评论需登录。

#### A12 孕周小能力

- **功能描述**：提供产检提醒（按孕周一句）、宝宝成长描述（按孕周 2～3 句）、本周提示（一句）等独立接口，供前端在首页、每日等位置展示。
- **使用流程**：前端在对应模块请求 `/api/ai/prenatalReminder`、`/api/ai/babyGrowth`、`/api/ai/weeklyTip`（可带 userId、week），后端按提示词模板调用 LLM 生成一句或几句文案并返回；前端直接展示。用户无输入，仅浏览。
- **业务规则**：宝宝发育描述禁止违禁比喻；产检提醒不替代医嘱。

### 2.4 家庭与共享

#### F01 家庭创建与加入

- **功能描述**：孕妇可创建家庭并获得邀请码与过期时间；其他用户通过邀请码加入家庭，并可选填与孕妇关系（如配偶、婆婆、妈妈）。
- **使用流程**：  
  1. 孕妇在「家庭」或「家人共享」中点击「创建家庭」，系统生成 family、邀请码与过期时间，并创建一条 family_member（creator）。  
  2. 孕妇将邀请码与有效期告知家人；家人在「加入家庭」页输入邀请码，可选填与孕妇关系后提交。  
  3. 系统校验邀请码与有效期，为该用户创建 family_member，若填了关系则写入 relationship，并可能异步调用配偶判断（LLM）写入 is_spouse。  
  4. 加入成功后，该用户可在「家人的记录」中按分享范围与可见范围查看记录。
- **业务规则**：邀请码唯一且有过期时间；同一用户不能重复加入同一家庭；关系为可选，但影响配偶权限与任务分配。

#### F02 成员管理

- **功能描述**：家庭创建者可查看成员列表、维护成员与孕妇关系、查看配偶标识；成员可主动离开；创建者可解散家庭。
- **使用流程**：  
  1. 创建者进入成员列表，查看各成员的 relationship、is_spouse（若已判断）。  
  2. 可编辑某成员的「与孕妇关系」，系统更新 relationship，并可能再次异步判断 is_spouse。  
  3. 成员本人可点击「离开家庭」，系统删除其 family_member；创建者可「解散家庭」，删除该 family 及所有 family_member。  
  4. 配偶标识用于开放 AI 对话、图生图等，前端根据 isSpouse 或关系关键词展示对应入口。
- **业务规则**：仅创建者可编辑关系与解散；离开或解散后，该家庭下的任务与通知等需一致处理。

#### F03 记录评论与点赞

- **功能描述**：家庭成员对可见的记录可发表评论（含回复他人）、对记录点赞、对评论点赞。
- **使用流程**：  
  1. 在家人记录详情或列表中，用户点击「评论」输入内容提交；系统写入 record_comment，可选 parent_comment_id 表示回复某条评论。  
  2. 用户点击「点赞」：系统写入或取消 record_like，并更新展示的点赞数。  
  3. 用户对某条评论点赞：系统写入或取消 record_comment_like。  
  4. 评论与点赞列表按时间或热度展示。
- **业务规则**：仅对当前用户有权限查看的记录可评论与点赞；删除记录时级联删除评论与点赞。

#### F04 家庭任务（爸爸成长营）

- **功能描述**：创建者可给成员（多为配偶）分配任务；系统可按孕周生成任务建议；成员可查看「我的任务」与「家庭任务」列表，并标记完成。
- **使用流程**：  
  1. 创建者进入爸爸成长营/家庭任务，可点击「生成本周建议」：系统根据孕周与已有任务调用 AI 生成 3～5 条建议（标题+描述），前端展示供选择。  
  2. 创建者选择部分建议并指定执行人（assignee），创建 family_task；被分配用户会收到站内通知。  
  3. 成员在「我的任务」中看到待办与已完成，可点击「完成」更新状态；创建者在「家庭任务」中查看全部成员任务。  
  4. 创建者也可手动新建任务（标题、描述、执行人、截止日期等）。
- **业务规则**：任务有 status（pending/completed）、task_type（routine/emotion）；完成时更新 completed_at。

#### F05 站内通知

- **功能描述**：用户收到任务分配、系统提醒等站内通知；可查看未读数量、列表、标记已读或全部已读；部分提醒（如「添加配偶」）仅推送一次。
- **使用流程**：  
  1. 用户登录后，前端轮询或拉取未读数量与列表；有未读时在导航或铃铛处展示红点或数字。  
  2. 用户进入通知列表，点击某条可跳转关联任务或页面，并标记该条已读；可一键「全部已读」。  
  3. 对于「邀请配偶加入」「添加配偶后可分配任务」等提醒，系统在插入前按 userId+type+title 查重，若已存在则不再插入，保证只提醒一次。
- **业务规则**：通知类型含 task_assigned、system 等；related_task_id 关联家庭任务；read_at 表示已读时间。

#### F06 配偶提及

- **功能描述**：当用户在 AI 对话中@配偶（如「跟老公说一声我明天产检」），系统可向配偶邮箱发送摘要邮件；若当前用户是配偶则向孕妇（家庭创建者）发邮件。
- **使用流程**：  
  1. 用户在对话中输入包含@配偶意图的内容并发送。  
  2. 后端在流式回复结束后检测是否提及配偶（如关键词或简单规则），若是且当前用户为孕妇则获取配偶邮箱，发送摘要邮件；若当前用户为配偶则向创建者邮箱发送。  
  3. 邮件内容包含发送者昵称与对话摘要；若配偶/孕妇未绑定邮箱则仅在回复中提示，不抛错。  
  4. 用户可在回复中看到「已发送给配偶」或「已发送给老婆」等提示。
- **业务规则**：邮件为通知性质，不替代站内消息；需已绑定邮箱才发送。

### 2.5 每日与健康

#### D01 每日日志

- **功能描述**：用户按日期记录当日胎动次数、心情、体重、健康值；可查看今日日志与历史曲线（如近 7 天心情、健康值）。

  2. 系统按 user_id + record_date 唯一写入或更新 user_daily_log。  
  3. 用户可查看历史曲线：选择「近 7 天」等，前端请求 moodHistory、healthHistory 等接口，绘制折线或列表。  
  4. 体重、健康值等可在记录时快照到 memo（若有），也可仅在每日日志中维护。
- **业务规则**：每个用户每天一条 user_daily_log；胎动为次数，心情为枚举（如 happy、calm、tired）。

#### D02 宫缩记录

- **功能描述**：用户按日期记录宫缩开始时间与持续时间；可查看某日列表，可清除某日全部记录。
- **使用流程**：  
  1. 用户进入宫缩记录页，选择日期后点击「添加」：输入开始时间与持续秒数，提交。  
  2. 系统写入 contraction 表。  
  3. 列表按日期展示该日所有宫缩记录；用户可「清除当日」删除该日全部记录。
- **业务规则**：数据按 userId + 日期过滤；清除为物理删除或软删视实现而定。

#### D03 产检单

- **功能描述**：用户上传产检 PDF，系统解析摘要与建议复查日期并写入 check_report；解析后的正文会写入向量库（source=pdf）；到期日前可发送邮件提醒。
- **使用流程**：  
  1. 用户进入产检单页，上传 PDF 文件，可附简短补充说明。  
  2. 系统上传文件至 OSS，使用 PDFBox 抽取正文，再调用 LLM 按 check_report_extract 提示词解析出 summary、next_check_date，写入 check_report；并将抽取正文嵌入向量库（source=pdf，source_id=reportId）。  
  3. 列表展示各条报告的摘要、建议复查日期、文件链接。  
  4. 定时任务在建议复查日临近或当日向用户邮箱发送提醒邮件（若用户已绑定邮箱且发送状态允许）。
- **业务规则**：仅 PDF 做正文抽取与向量嵌入；非 PDF 不解析；邮件发送失败会更新 send_status 并可能重试。

- **业务规则**：同一 badge_key 可限制每人仅获得一次，视业务而定。

### 2.8 前端与体验

#### E01 聊天页

- **功能描述**：空状态展示暖心语与快捷问题；消息气泡带轻量进入动画；流式回复时尾部有脉冲光标；快捷按钮与发送按钮有 hover/过渡效果。
- **使用流程**：用户进入聊天页即看到上述交互；发送消息后气泡以动画出现，AI 回复流式展示并带光标，结束即停；点击快捷问题自动填入并发送。
- **业务规则**：暖心语仅无消息时展示；超时后发送按钮恢复可点。

#### E02 时间轴页

- **功能描述**：日期分组块淡入；标签与分类展示时超过 6 字截断并省略号；筛选与搜索交互清晰。
- **使用流程**：用户进入时间轴即看到分组与列表动效；筛选与搜索即时生效；标签展示统一≤6 字。
- **业务规则**：与 M05 一致。

#### E03 验证码

- **功能描述**：登录/注册时展示图形验证码，用户填写后与服务端校验，通过才继续登录/注册。
- **使用流程**：用户打开登录/注册页，获取验证码图片并输入框中填写，提交时服务端校验验证码与账号密码；验证码错误时提示重新获取并输入。
- **业务规则**：验证码有效期与一次性使用由服务端控制。

## 三、非功能需求摘要

- **可用性**：Redis 不可用时仅打日志降级，不中断业务；MySQL 连接异常时定时任务单轮跳过并重试。
- **安全**：密码加密存储；会话与记录按用户/家庭鉴权。
- **性能**：用户信息等热点数据使用 Redis 缓存，TTL 5–10 分钟；RAG 检索 top_k 可配（默认 10）。
- **可维护性**：AI 提示词统一存储在 `prompt_template` 与 JSON 配置中，便于迭代。

以上功能列表与使用流程基于当前已实现功能整理，用于产品与评审对齐。
