# 孕期宝 功能文档

## 一、项目概述

### 1.1 项目简介

**孕期宝**是一款面向孕妇及家庭的孕期记录与陪伴应用。通过文字、语音、照片、文件等多种形式记录孕期点滴，结合 AI 智能陪伴、家庭协同、健康追踪等功能，为用户提供全方位的孕期服务。

### 1.2 目标用户

| 用户角色 | 说明 | 核心功能权限 |
|---------|------|-------------|
| 孕妇（主账号） | 创建家庭的核心用户 | 全量记录、AI 对话、图生图、目标与成就、产检单管理、分享范围设置 |
| 家庭成员 | 通过邀请码加入家庭 | 查看共享记录、评论点赞、家庭任务、配偶可使用 AI 对话与图生图 |
| 管理员 | 后台管理系统 | 用户与内容管理、科普文章发布、数据统计 |

## 二、功能模块详解

### 2.1 用户与账号模块

#### 2.1.1 注册与登录

| 功能点 | 说明 |
|--------|------|
| 用户注册 | 用户名、密码、确认密码、与孕妇关系（可选） |
| 用户登录 | 用户名、密码、图形验证码（可选开启） |
| 密码找回 | 通过绑定邮箱发送重置链接 |

**业务规则：**
- 密码加密存储（哈希算法）
- 验证码服务端校验，有效期内有效
- 用户类型区分：孕妇(pregnant) / 家庭成员(family_member)

#### 2.1.2 个人资料管理

| 功能点 | 说明 |
|--------|------|
| 头像设置 | 上传图片至 OSS |
| 昵称修改 | 用户名展示 |
| 邮箱绑定 | 用于接收提醒、找回密码 |
| 密码修改 | 需验证当前密码 |
| 孕产信息 | 末次月经日、预产期（孕妇必填） |

#### 2.1.3 隐私与分享设置

| 设置项 | 选项 |
|--------|------|
| 匿名数据收集 | 开启/关闭（默认关闭） |
| 分享范围 | 全部 / 仅信件 / 仅照片 |

### 2.2 孕期记录模块（备忘录）

#### 2.2.1 记录类型

| 类型 | 支持内容 | 特殊功能 |
|------|----------|----------|
| **文字记录** | 标题、正文、孕周、标签、心情、可见范围、分类 | AI 生成标题与分类 |
| **语音记录** | 语音文件、标题、孕周、心情、可见范围、分类 | AI 生成标题 |
| **照片记录** | 1-9张照片、标题、描述、孕周、心情、可见范围、分类 | 图像理解+AI描述 |
| **文件记录** | 文档/视频/PDF等、标题、孕周、心情、可见范围、分类 | PDF正文抽取嵌入向量库 |

#### 2.2.2 可见范围控制

| 模式 | 说明 |
|------|------|
| 全部可见 | 所有家庭成员可见 |
| 仅指定成员可见(Allowlist) | 仅勾选成员可见，空则仅自己 |
| 仅指定成员不可见(Blocklist) | 除勾选成员外可见，空则全部可见 |

#### 2.2.3 时间轴展示

- 按日期分组倒序展示
- 支持按类型筛选：全部、重要时刻、给未来的自己、文字、照片、语音、文件
- 关键词搜索（标题、内容、描述、标签）
- 标签展示限6字，超出截断显示省略号

#### 2.2.4 导出功能

- **导出全部**：生成 PDF 纪念册（含封面、孕周分组、结语）
- **按日期范围导出**：选择起止日期导出该时间段记录

### 2.3 AI 对话模块

#### 2.3.1 会话管理

| 功能 | 说明 |
|------|------|
| 创建会话 | 开始新对话 |
| 会话列表 | 展示历史会话（标题、更新时间） |
| 删除会话 | 删除会话及所有消息 |
| 加载历史 | 按时间序展示会话消息 |

#### 2.3.2 流式对话

- **SSE 流式输出**：AI 回复逐字显示
- **多轮对话**：自动带入近期历史上下文
- **多模态支持**：文本 + 图片输入
- **超时控制**：2分钟超时自动恢复

#### 2.3.3 意图识别分支

```
用户输入
    │
    ▼
┌────────────────────────────────────────┐
│           意图识别                      │
└────────────────────────────────────────┘
    │
    ├───▶ REMINDER（定时提醒）
    │      └── 多轮收集时间/地点/内容 → 写入 scheduled_operation
    │
    ├───▶ TEXT_TO_IMAGE / IMAGE_TO_IMAGE（图生图）
    │      └── 调用图像生成 API
    │
    ├───▶ IMAGE_UNDERSTANDING（图片理解）
    │      └── 图像理解后走 RAG 或普通对话
    │
    └───▶ TEXT_CHAT（普通对话）
           └── RAG 检索 → 大模型生成回复
```

#### 2.3.4 RAG 检索增强

- **向量检索**：用户问题向量化后检索 top_k=10 相关记录
- **上下文注入**：检索结果拼入 system prompt
- **图片保留**：检索结果中的图片 URL 保留 Markdown 语法
- **检索不嵌入**：检索轮次不将用户问题写入向量库（避免污染）

#### 2.3.5 AI 特色能力

| 能力 | 触发条件 | 功能说明 |
|------|----------|----------|
| **写信给宝宝** | 关键词：写信+宝宝/给宝宝/对宝宝 | 提供写信结构建议与润色 |
| **情绪共情** | 关键词：累、烦、担心、开心、紧张等 | 先共情再给建议 |
| **每日暖心语** | 聊天页空状态时自动请求 | 基于孕周与日期的鼓励语 |
| **本周小结** | 用户主动请求 | 总结本周记录与状态 |
| **找图/搜记录** | 关键词：找、搜、查+图片/记录 | 基于向量检索查找历史记录 |

#### 2.3.6 配偶端支持

当用户被识别为配偶时：
- 使用配偶专属 system prompt
- 侧重孕期知识、照顾妻子技巧、情感支持话术
- 提供小惊喜建议（录语音、写信给宝宝等）

#### 2.3.7 定时提醒

| 功能 | 说明 |
|------|------|
| 自然语言添加 | 「明天下午3点去人民公园散步」 |
| 多轮收集 | 最多2轮，收集时间、地点、做什么、备注 |
| 提醒类型 | 一次性(once) / 每天(daily) |
| 到期触发 | 站内通知 + 邮件提醒 |
| 文案改写 | 触发时 LLM 改写为当前场景话术 |

## 三、核心业务规则

### 3.1 可见性规则

```
家庭成员能否看到某条记录 =
    账号级分享范围(user.share_scope)
    AND
    记录级可见模式(memo.visibility_mode)
    AND
    记录级可见成员(memo.visible_to)
```

### 3.2 权限规则

| 功能 | 孕妇 | 配偶 | 其他家庭成员 |
|------|------|------|-------------|
| 创建记录 | ✅ | ❌ | ❌ |
| 编辑/删除记录 | ✅(自己) | ❌ | ❌ |
| AI 对话 | ✅ | ✅ | ❌ |
| 图生图 | ✅ | ✅ | ❌ |
| 查看家人记录 | ✅ | ✅(受范围限制) | ✅(受范围限制) |
| 评论点赞 | ✅ | ✅ | ✅ |
| 家庭任务 | ✅(分配) | ✅(执行) | ✅(执行) |
| 情景演绎 | ❌ | ✅ | ❌ |

### 3.3 数据嵌入向量库规则

| 数据类型 | 嵌入内容 | 来源 |
|----------|----------|------|
| 文字记录 | 标题+正文+[URL] | memo |
| 语音记录 | 标题+[URL] | memo |
| 照片记录 | 描述+图像理解结果+[URL] | image_desc |
| 文件记录 | 标题+[URL] | memo |
| PDF 文件 | 抽取正文+[URL] | memo/pdf |
| 对话消息 | 消息内容 | message |
| 产检报告 | 抽取正文 | pdf |

### 4.2 流式对话接口

```
POST /api/ai/chat-stream
Content-Type: multipart/form-data

参数：
- userId: 用户ID
- conversationId: 会话ID（可选，无则创建新会话）
- message: 用户消息
- images: 图片文件（可选，多图）

响应：SSE 流
- event: message
- data: {"content": "AI回复片段"}

结束标志：
- event: done
- data: {"messageId": 123}
```
