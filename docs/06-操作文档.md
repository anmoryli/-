# 孕期宝 操作文档

## 一、环境准备

### 1.1 系统要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 10 / Linux / macOS | Windows 11 / Ubuntu 20.04+ |
| Java | JDK 17 | JDK 17 LTS |
| Node.js | 18.x | 20.x LTS |
| Python | 3.10+ | 3.11 |
| MySQL | 8.0 | 8.0 |
| Redis | 6.0 | 7.0 |
| Milvus | 2.x | 2.3+ |

### 1.2 必需软件安装

#### 1.2.1 Java 环境

```bash
# 检查 Java 版本
java -version

# 应显示类似输出：
# openjdk version "17.0.x" 202x-xx-xx
# OpenJDK Runtime Environment (build 17.0.x)
```

#### 1.2.2 Node.js 环境

```bash
# 检查 Node 版本
node -v
npm -v

# 应显示：
# v20.x.x
# 10.x.x
```

#### 1.2.3 Python 环境

```bash
# 检查 Python 版本
python --version
# 或
python3 --version

# 应显示 Python 3.10.x 或更高
```

#### 1.2.4 MySQL 安装

```bash
# Windows (使用 MySQL Installer)
# 下载地址：https://dev.mysql.com/downloads/installer/
# 安装时设置 root 密码，记住密码

# 创建数据库
mysql -u root -p
CREATE DATABASE yunfu CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

#### 1.2.5 Redis 安装

```bash
# Windows (使用 Memurai 或 WSL)
# Memurai 下载：https://www.memurai.com/
# 或使用 WSL 安装 Redis

# 启动 Redis
redis-server
```

#### 1.2.6 Milvus 安装

```bash
# 使用 Docker Compose 安装 Milvus
# 参考官方文档：https://milvus.io/docs/install_standalone-docker.md

# 下载 docker-compose.yml
wget https://github.com/milvus-io/milvus/releases/download/v2.3.3/milvus-standalone-docker-compose.yml -O docker-compose.yml

# 启动 Milvus
docker-compose up -d
```

## 二、项目配置

### 2.1 后端配置 (Spring Boot)

#### 2.1.1 配置文件位置

```
backend/src/main/resources/
├── application.yml          # 主配置
├── application-dev.yml      # 开发环境配置
└── application-prod.yml     # 生产环境配置
```

#### 2.1.2 数据库配置

```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/yunfu?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&connectTimeout=15000&socketTimeout=30000
    username: root
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      connection-timeout: 20000
      initialization-fail-timeout: -1
```

#### 2.1.3 Redis 配置

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password:      # 无密码留空
      database: 0
      timeout: 2000ms
```

#### 2.1.4 AI 配置

```yaml
spring:
  ai:
    openai:
      base-url: https://api.openai.com/v1  # 或其他兼容接口地址
      api-key: sk-your-api-key
      chat:
        options:
          model: gpt-4
          temperature: 0.7
```

#### 2.1.5 OSS 配置

```yaml
aliyun:
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    accessKeyId: your-access-key
    accessKeySecret: your-access-secret
    bucketName: your-bucket-name
```

#### 2.1.6 RAG 服务配置

```yaml
rag:
  service:
    url: http://localhost:8000  # Python RAG 服务地址
```

#### 2.1.7 邮件配置

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: your-app-password
    protocol: smtp
    default-encoding: UTF-8
    properties:
      mail.smtp.auth: true

```

## 四、启动运行

### 4.1 启动顺序

```
1. MySQL      → 确保数据库运行
2. Redis      → 确保缓存服务运行
3. Milvus     → 确保向量库运行
4. RAG服务    → Python 嵌入/检索服务
5. 后端服务   → Spring Boot 应用
6. 前端服务   → Next.js 开发服务器
7. 管理后台   → Flask 应用（可选）
```

### 4.2 启动 RAG 服务

```bash
# 进入 RAG 服务目录
cd backend/src/main/resources/py

# 安装依赖（首次）
pip install fastapi uvicorn pymilvus requests

# 启动服务
python app.py

# 或使用 uvicorn
uvicorn app:app --host 0.0.0.0 --port 8000 --reload

# 服务地址：http://localhost:8000
```

### 4.3 启动后端服务

```bash
# 进入后端目录
cd backend

# Maven 编译
mvn clean compile

# 运行（开发环境）
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 或打包后运行
mvn clean package -DskipTests
java -jar target/yunqibao.jar --spring.profiles.active=dev

# 服务地址：http://localhost:9677
```

### 4.4 启动前端服务

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次）
npm install
# 或
pnpm install

# 启动开发服务器
npm run dev
# 或
pnpm dev

# 访问地址：http://localhost:3000
```

### 4.5 启动管理后台

```bash
# 进入管理后台目录
cd admin-app

# 安装依赖（首次）
pip install -r requirements.txt

# 启动服务
python app.py

# 或使用 Flask 命令
flask run --host=0.0.0.0 --port=5000

# 访问地址：http://localhost:5000
# 如果设置了 ADMIN_PASSWORD，会要求输入密码
```

## 五、功能操作指南

### 5.1 用户端操作流程

#### 5.1.1 注册与登录

1. **访问首页** → http://localhost:3000
2. **点击注册** → 填写用户名、密码、确认密码
3. **选择身份** → 孕妇 / 家庭成员
4. **完善资料** → 孕妇需填写末次月经日和预产期
5. **登录系统** → 使用用户名和密码登录

#### 5.1.2 创建记录

**文字记录：**
1. 点击「写日记」
2. 填写标题（可选，可 AI 生成）
3. 编写正文内容
4. 选择孕周、心情、标签
5. 设置可见范围
6. 点击保存

**照片记录：**
1. 点击「拍照片」
2. 选择 1-9 张照片上传
3. 填写标题和描述
4. 系统会自动进行图像理解
5. 点击保存

**语音记录：**
1. 点击「录语音」
2. 录制或选择语音文件
3. 填写标题（可选）
4. 点击保存

**文件记录：**
1. 点击「上传文件」
2. 选择文件（支持 PDF、视频、文档等，最大 500MB）
3. 填写标题
4. 点击保存

#### 5.1.3 AI 对话

1. **进入聊天页** → 点击底部导航「AI助手」
2. **开始对话** → 输入文字或上传图片
3. **使用快捷问题** → 点击预设问题快速发送
4. **查看历史** → 侧边栏展示历史会话
5. **创建新会话** → 点击「新对话」按钮

**特色功能触发：**
- 「帮我写一封信给宝宝」→ 触发写信引导
- 「找一下我之前拍的玉桂狗照片」→ 触发图片检索
- 「明天下午3点提醒我去公园散步」→ 触发定时提醒

#### 5.1.4 家庭管理

**创建家庭：**
1. 进入「家庭」页面
2. 点击「创建家庭」
3. 系统生成邀请码和过期时间
4. 将邀请码分享给家人

**加入家庭：**
1. 点击「加入家庭」
2. 输入邀请码
3. 选择/填写与孕妇的关系
4. 点击确认加入

**查看家人记录：**
1. 进入「家人的记录」
2. 按类型筛选查看
3. 对记录评论、点赞

#### 5.1.5 每日记录

1. **进入每日页面**
2. **记录胎动** → 点击计数或手动输入
3. **选择心情** → 点击心情图标
4. **记录体重** → 输入当日体重
5. **健康自评** → 滑动选择 0-100 分
6. **保存记录**

#### 5.1.6 产检单管理

1. **进入产检单页面**
2. **点击上传** → 选择 PDF 文件
3. **填写补充说明**（可选）
4. **等待 AI 解析** → 自动提取摘要和复查日期
5. **查看报告列表** → 显示摘要和复查提醒

#### 5.1.7 图生图

1. **进入图生图页面**
2. **选择模板**（可选）或自由输入
3. **上传基础图片**
4. **输入提示词**
5. **选择是否公开到社区**
6. **点击生成** → 等待 AI 生成图片
7. **查看结果** → 保存或重新生成

#### 5.1.8 目标与成就

1. **进入目标页面**
2. **查看当前进度** → 各目标完成百分比
3. **完成任务** → 系统自动更新进度
4. **查看成就** → 已获得徽章列表

## 六、常见问题排查

### 6.1 后端启动问题

**问题：端口被占用**
```bash
# 查找占用 9677 端口的进程
netstat -ano | findstr :9677
# 或 Linux
lsof -i :9677

# 结束进程或修改后端端口
# application.yml 中修改 server.port
```

**问题：数据库连接失败**
```bash
# 检查 MySQL 是否运行
mysql -u root -p -e "SELECT 1"

# 检查配置文件中密码是否正确
# 检查数据库 yunfu 是否存在
```

**问题：Redis 连接失败**
```bash
# 检查 Redis 是否运行
redis-cli ping
# 应返回 PONG

# 如 Redis 未配置密码，确保配置文件中的 password 为空
```

### 6.2 前端启动问题

**问题：依赖安装失败**
```bash
# 清理缓存后重试
rm -rf node_modules package-lock.json
npm install

# 或使用 pnpm
pnpm install
```

**问题：代理接口超时**
```bash
# 检查后端服务是否正常运行
curl http://localhost:9677/api/health

# 修改 .env.local 直接指向后端
NEXT_PUBLIC_API_BASE_URL=http://localhost:9677
```

### 6.3 RAG 服务问题

**问题：Milvus 连接失败**
```bash
# 检查 Milvus 是否运行
docker ps | grep milvus

# 检查 Milvus 端口是否可连接
telnet localhost 19530
```

**问题：嵌入失败**
```bash
# 检查 Embedding API 配置是否正确
# 查看 RAG 服务日志中的错误信息
```

### 6.4 AI 对话问题

**问题：流式输出不流畅**
- 检查网络连接
- 检查大模型 API 响应速度
- 查看前端控制台是否有错误

**问题：检索不到历史记录**
- 确认记录已写入向量库
- 检查 RAG 服务是否正常运行
- 查看后端日志中的 RAG 调用情况
